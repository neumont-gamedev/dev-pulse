# Dev Pulse - Project Overview

## Vision

Dev Pulse is a project management and development tracking platform designed specifically for students working on long-term software, game development, and capstone projects.

The goal of Dev Pulse is to encourage consistent progress, increase accountability, improve communication between students and instructors, and provide visibility into project health throughout the development process.

Unlike traditional project submissions that only evaluate final deliverables, Dev Pulse focuses on the entire development journey by tracking work completed, milestones achieved, obstacles encountered, and overall project momentum.

The system helps students develop professional development habits while giving instructors better insight into how projects are progressing between milestones and presentations.

---

## Problem Statement

Many students procrastinate and work in large bursts immediately before project milestones, check-ins, or final presentations.

This often leads to:

* Incomplete projects
* Lower quality work
* Poor planning and scope management
* Difficulty identifying struggling students
* Limited visibility into the actual development process

Instructors typically only see project progress during scheduled check-ins, making it difficult to identify issues early.

Dev Pulse addresses this problem by creating a continuous development tracking system that encourages regular updates and project reflection.

---

## Primary Goals

### Student Goals

* Track development progress over time
* Log work completed and hours invested
* Record blockers and challenges
* Manage project milestones
* Maintain a professional development journal
* View project health and momentum
* Prepare for milestone reviews and presentations

### Instructor Goals

* Monitor project activity across all teams
* Identify inactive or struggling projects
* Review milestone readiness
* Track student engagement
* View project health metrics
* Receive milestone review requests
* Provide feedback throughout development

---

## User Roles

### Student

Students can:

* Create and manage projects
* Join project teams
* Submit development log entries
* Record hours worked
* Track progress toward milestones
* Upload links to builds, videos, screenshots, and repositories
* Request milestone reviews
* View project analytics and history

### Instructor

Instructors can:

* View all projects
* Review development activity
* Approve or reject milestone submissions
* Monitor team engagement
* Identify projects at risk
* View project summaries
* Leave feedback on milestones and progress reports

---

## Core Features

### Project Creation

Students create a project and provide:

* Project name
* Project description
* Team members
* Development platform
* Game engine or technology stack
* GitHub repository URL
* Build URL (optional)

---

### Development Log System

Students create development log entries whenever meaningful progress is made.

Each entry includes:

* Date
* Hours worked
* Work completed
* Problems encountered
* Blockers
* Next steps
* Build links
* Screenshots or evidence

The development log becomes a timeline showing the evolution of the project.

---

### Hours Tracking

Students record time spent on project work.

Examples:

* Programming
* Design
* Art
* Audio
* Research
* Testing
* Documentation
* Team meetings

The system displays:

* Weekly hours
* Total project hours
* Team contribution statistics

---

### Milestone Tracking

Projects progress through predefined milestones.

Example milestones:

* Project Proposal
* Prototype
* Vertical Slice
* Feature Complete
* Playtest Ready
* Polish Pass
* Final Build
* Final Presentation

Milestone states include:

* Not Started
* In Progress
* Ready for Review
* Approved
* Needs Revision

When students mark a milestone as Ready for Review, instructors receive a notification that the project is ready for a check-in.

---

### Instructor Review Workflow

Students submit milestone reviews.

Submission requirements may include:

* Build link
* Screenshot
* Video demonstration
* Development notes

The instructor can:

* Approve
* Request revisions
* Leave feedback

The milestone history becomes part of the project's permanent record.

---

## AI Integration

Artificial Intelligence is intended to assist students and instructors, not replace development work.

### Student AI Features

* Generate devlog drafts from notes
* Summarize recent work
* Organize progress reports
* Generate milestone summaries
* Suggest next steps

### Instructor AI Features

* Project health summaries
* Activity analysis
* Risk identification
* Weekly project overviews
* Team progress reports

All AI-generated content should be stored separately from student-authored content.

---

## Project Health Dashboard

Each project has a dashboard displaying:

* Current milestone
* Recent activity
* Total hours worked
* Team participation
* Last update date
* Project health score

Example indicators:

* Active
* At Risk
* Behind Schedule
* On Track
* Ready for Review

### Notifications

* Milestones are ready for review
* Projects become inactive

---

## Educational Philosophy

Dev Pulse is not intended to function as surveillance software.

The purpose of the platform is to:

* Encourage consistent development habits
* Promote project planning
* Improve communication
* Support reflection and learning
* Simulate professional development workflows

The platform should help students become better developers by focusing on the process of building software rather than only evaluating the final result.

---

## Minimum Viable Product (MVP)

The first release should include:

1. User authentication
2. Student project creation
3. Development log entries
4. Hours tracking
5. Milestone management
6. Instructor dashboard
7. Milestone review workflow
8. Basic AI-generated summaries

Advanced GitHub integration and analytics can be added in later versions.
